Week 10/05 I have created the filter panel that can be hided when needed. Then the hurricanes have been inserted in form of a list that can be scrolled. The hurricanes can be selected (one or more) in order to hide or show it on the map and the filters to show all of them at once. The list shows also the attribute speed and date. The landed filter has been added in order to select only those hurricans that landed. The calendar to select the period to filter the hurricanes has been implemented but not integrated yet. Transferred diagrams from paper to digital. I have created a script that extracts some statistics from the dataset and format them in a JSON object. I have created a left side panel scrollable composed by three subpanels that will contains the graphs relative to: 1- Overall hurricanes statistics, i.e. graphs visualizing data from the entire dataset. 2- Visualized hurricanes statistics, i.e graphs visualizing data about hurricanes visualized on the map. 3 - Selected hurricanes statistics, i.e graphs visualizing data about one selected hurricane. I have created some functions that generates the graphs relative to: 1- Number of hurricanes per year (BarChart) 2- Max Wind Speed per year (Line Chart, with circles in points for clarity) 3- Minimum Pressure per year (Line Chart, with circles in points for clarity, the black circles mean that the data wasn't available for that year, and i levelled all of them to 900). Implemented filtering functions that allow the user to filter the data visualized on the map (e.g. select hurricanes belonging to an interval of dates, get the top 10 hurricanes related to, for instance, max wind speed etc...). Implemented the model that provides the above functionalities to the visual part of the application, using the observer paradigm to update the map and the other visual tabs.
